  ## Install the core package and an adapter

  The `@unbook/core` package provides the unified API, while an adapter (like `@unbook/notion`) handles communication with a specific platform.

  ::code-group
  ```bash [pnpm]
  pnpm add @unbook/core @unbook/notion
  ```

  ```bash [npm]
  npm install @unbook/core @unbook/notion
  ```

  ```bash [yarn]
  yarn add @unbook/core @unbook/notion
  ```

  ```bash [bun]
  bun add @unbook/core @unbook/notion
  ```
  ::


  ## Set up the workspace

  Instantiate the adapter and create a `Workspace` instance. This is your main entry point for interacting with the API and entities.

  ::tabs{.w-full}
  :::tabs-item{icon="i-lucide-user" label="Get Current User"}
  ```ts [get-current-user.ts]
  import { Workspace } from '@unbook/core'
  import { Notion } from '@unbook/notion'

  // 1. Instantiate the Notion adapter and workspace
  const adapter = new Notion({ token: process.env.NOTION_TOKEN })
  const workspace = new Workspace(adapter)

  // 2. Get the user associated with the token
  const currentUser = await workspace.getCurrentUser()
  ```
  :::

  :::tabs-item{icon="i-lucide-search" label="Search Pages"}
  ```ts [search-pages.ts]
  import { Workspace } from '@unbook/core'
  import { Notion } from '@unbook/notion'

  // 1. Instantiate the Notion adapter and workspace
  const adapter = new Notion({ token: process.env.NOTION_TOKEN })
  const workspace = new Workspace(adapter)

  // 2. Search for pages with "project" in the title or content
  const pages = workspace.search('project', { filter: 'page' })

  // 3. Iterate over search results
  for await (const page of pages) {
    console.log(page.title, page.url)
  }
  ```
  :::

  :::tabs-item{icon="i-lucide-file-text" label="Render Markdown"}
  ```ts [render-markdown.ts]
  import { Workspace } from '@unbook/core'
  import { Notion } from '@unbook/notion'

  // 1. Instantiate the Notion adapter and workspace
  const adapter = new Notion({ token: process.env.NOTION_TOKEN })
  const workspace = new Workspace(adapter)

  // 2. Get a page by ID
  const page = await workspace.getPage('page-id-here')

  // 3. Render its blocks to plain text
  const markdown = await page.render()
  console.log(markdown)
  ```
  :::
  ::

  ::callout{icon="i-lucide-info"}
  Get your token at [Notion Integrations](https://www.notion.so/my-integrations).
  ::

  ## Create a page

  Use `workspace.createPage()` to create a new page. You can set the title, icon, and cover.

  ::tabs{.w-full}
  :::tabs-item{icon="i-lucide-file" label="Root Page"}
  ```ts [root-page.ts]
  // 1. Create a new page at the root level
  const page = await workspace.createPage('My First Page')
  ```
  :::

  :::tabs-item{icon="i-lucide-folder" label="Subpage"}
  ```ts [subpage.ts]
  // 1. Get the parent page
  const parentPage = await workspace.getPage('parent-page-id')

  // 2. Create a subpage under the parent
  const subPage = await workspace.createPage('Subpage', { parent: parentPage })
  ```
  :::

  :::tabs-item{icon="i-lucide-palette" label="Styled"}
  ```ts [styled.ts]
  // 1. Create a page with custom icon and cover
  const styledPage = await workspace.createPage('Styled Page', {
    icon: 'ðŸ“„',
    cover: 'https://example.com/cover.jpg'
  })
  ```
  :::

  :::tabs-item{icon="i-lucide-file-plus" label="With Content"}
  ```ts [content.ts]
  import { blocks } from '@unbook/core'

  // 1. Create a page with initial content blocks
  const contentPage = await workspace.createPage('Content Page', {}, [
    blocks.heading1('Welcome to Unbook'),
    blocks.paragraph('This is a sample page created with Unbook.')
  ])
  ```
  :::
  ::


  ::callout{icon="i-lucide-info"}
  Pages are the fundamental building blocks of your workspace. They can contain text, blocks, and other content, and can be organized hierarchically. For more details, see the [Pages guide](/guides/pages).
  ::

  ## Query a database

  Use `workspace.ensureDatabase()` to create or synchronize a database with a defined schema, then query its records. Use `fields` shorthands to define your schema declaratively.

  ::tabs{.w-full}
  :::tabs-item{icon="i-lucide-database" label="Create Database"}
  ```ts [create-database.ts]
  import { fields } from '@unbook/core'

  // 1. Create a page to host the database
  const page = await workspace.createPage('Tasks Page')

  // 2. Get or create a database with a schema
  const tasks = await workspace.ensureDatabase('Tasks', {
    parent: page,
    schema: {
      name: fields.title('Name'),
      status: fields.select('Status', ['Todo', 'In Progress', 'Done']),
      priority: fields.select('Priority', ['Low', 'Medium', 'High']),
      dueDate: fields.date('Due Date')
    }
  })
  ```
  :::

  :::tabs-item{icon="i-lucide-plus" label="Insert Records"}
  ```ts [insert-records.ts]
  // 1. Upsert a record (update if exists, insert if not)
  await tasks.upsert(['name'], { name: 'Review docs', priority: 'High' })

  // 2. Insert a new record
  await tasks.insert({ name: 'Write tests', status: 'Todo', dueDate: new Date() })
  ```
  :::

  :::tabs-item{icon="i-lucide-filter" label="Query Records"}
  ```ts [query-records.ts]
  // 1. Find records with filters and sorting
  const iterator = tasks.find({
    filter: { status: 'In Progress' },
    sort: { dueDate: 'asc' }
  })

  // 2. Iterate over the results
  for await (const record of iterator) {
    console.log(record.name, record.status)
  }
  ```
  :::
  ::

  ::callout{icon="i-lucide-info"}
  Databases store data in structured tables called data sources. Each data source has a schema defining its properties (columns) and their types. For more details, see the [Databases guide](/guides/databases).
  ::

  ## Add blocks to a page

  Pages contain blocks, which are the content elements like text, headings, lists, media, and more. Use the `blocks` shorthands to create blocks easily.

  ```ts [src/index.ts]
  import { blocks } from '@unbook/core'

  // 1. Get an existing page
  const page = await workspace.getPage('page-id')

  // 2. Clear existing content
  await page.clear()

  // 3. Append blocks to the page
  await page.append([
    blocks.heading1('Welcome to Unbook'),
    blocks.paragraph('This is a paragraph with some text.'),
    blocks.heading2('Features'),
    blocks.bulletedListItem('Unified API for multiple platforms'),
    blocks.bulletedListItem('Type-safe entities and operations'),
    blocks.bulletedListItem('Declarative schema management'),
    blocks.divider(),
    blocks.callout('Pro tip: Use shorthand builders for cleaner code!', { icon: 'ðŸ’¡' })
  ])
  ```

  ::callout{icon="i-lucide-info"}
  Blocks are the building blocks of page content. They can be nested to create complex layouts. For more details, see the [Blocks guide](/guides/blocks).
