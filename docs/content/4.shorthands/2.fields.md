Field shorthand functions simplify the definition of database schema fields. Instead of manually constructing field definition objects, use these helpers for cleaner, more readable schema definitions. Field shorthands are exported as a namespace from `@unbook/core/shorthands`:

::tabs{.w-full}
:::tabs-item{icon="i-lucide-code" label="Usage"}
```ts
import { fields } from '@unbook/core/shorthands'
fields.title('Task Name')
fields.select('Status', ['To Do', 'Done'])
// and more...
```
:::
:::tabs-item{icon="i-lucide-database" label="With Schema"}
```ts
const parentPage = await workspace.getPage('page-id')
const dataSource = await workspace.ensureDatabase({
  title: 'Project Tasks',
  parent: parentPage,
  schema: {
    name: fields.title('Task'),
    status: fields.select('Status', ['Backlog', 'In Progress', 'Review', 'Done']),
    dueDate: fields.date('Due Date'),
    assignee: fields.people('Assignee')
  }
})
```
:::
::

:::accordion-item{label="Select" icon="i-lucide-chevron-down-circle"}
::::tabs{.w-full}
:::::tabs-item{label="Simple" icon="i-lucide-file-text"}
When using the simple format, the key of each option is derived from its label.
```ts
fields.select('Priority', ['Low', 'Medium', 'High'])
```
:::::
:::::tabs-item{label="Keyed" icon="i-lucide-sliders-horizontal"}
When using the keyed format, the key of each option stays consistent even if the label changes.

```ts
fields.select('Priority', {
  low: { label: 'Low', color: 'green' },
  medium: { label: 'Medium', color: 'yellow' },
  high: { label: 'High', color: 'red' }
})
```
:::::
::::
:::

:::accordion-item{label="Multi-Select" icon="i-lucide-list-checks"}
::::tabs{.w-full}
:::::tabs-item{label="Simple" icon="i-lucide-file-text"}
When using the simple format, the key of each option is derived from its label.
```ts
fields.multiSelect('Tags', ['Frontend', 'Backend', 'DevOps', 'Design'])
```
:::::
:::::tabs-item{label="With options" icon="i-lucide-sliders-horizontal"}
When using the keyed format, the key of each option stays consistent even if the label changes.
```ts
fields.multiSelect('Labels', {
  frontend: { label: 'Frontend', color: 'blue' },
  backend: { label: 'Backend', color: 'green' },
  devops: { label: 'DevOps', color: 'orange' },
  design: { label: 'Design', color: 'pink' }
})
```

:::accordion-item{label="Formula" icon="i-lucide-function-square"}
```ts
fields.formula('Total', 'prop("Price") * prop("Quantity")')
```

::::::note{icon="i-lucide-info"}
Since labels can change over time, this library allows you to reference the properties by their keys in formula expressions. Internally, Unbook will convert these to the correct labels when querying the database. See the [Build filter expressions](../6.hydration.md#build-filter-expressions) section.
<br><br>
If the input expression references non-existent keys, they will be left unchanged and may lead to API errors if they do not match any property labels.
::::::
