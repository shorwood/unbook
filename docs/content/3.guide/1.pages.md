Pages are the fundamental building blocks of your workspace. They can contain text, blocks, and other content, and can be organized hierarchically.

## Create a page

Use `workspace.createPage()` to create a new page. You can specify a title, icon, cover, and parent.

```ts [create-page.ts]
import { createNotionAdapter } from '@unbook/notion'
import { Workspace } from '@unbook/core'

const adapter = createNotionAdapter({ auth: process.env.NOTION_TOKEN })
const workspace = new Workspace(adapter)

// Create a simple page
const page = await workspace.createPage('My New Page')
console.log(page.id, page.url)
```

### Create a page with options

You can customize the page with an icon, cover, and parent:


### Create a page with initial content

Pass an array of block inputs as the third argument to add content immediately:


## Retrieve a page

Use `workspace.getPage()` to fetch a page by its ID:

```ts [get-page.ts]
const page = await workspace.getPage('page-id-here')

// Access page properties
console.log('ID:', page.id)
console.log('Title:', page.title)
console.log('URL:', page.url)
console.log('Created:', page.createdAt)
console.log('Last edited:', page.lastEditedAt)
```

## Navigate page relationships

Pages exist within a hierarchy. You can navigate to parent entities and child blocks.

### Get the parent

A page's parent can be another Page, a Block, or a DataSource:

```ts [get-parent.ts]
const parent = await page.getParent()

if (parent instanceof Page) {
  console.log('Parent page:', parent.title)
} else if (parent instanceof DataSource) {
  console.log('Parent data source:', parent.title)
} else if (parent instanceof Block) {
  console.log('Parent block:', parent.type)
} else {
  console.log('Page is at workspace root')
}
```

### Get child blocks

Iterate over the page's content blocks:

```ts [get-children.ts]
for await (const block of page.getChildren()) {
  console.log(block.type, block.id)
}
```

### Get creator and last editor

```ts [get-users.ts]
const creator = await page.getCreator()
console.log('Created by:', creator.name)

const editor = await page.getLastEditor()
console.log('Last edited by:', editor.name)
```

## Update page properties

Use `page.update()` to modify page properties:

```ts [update-page.ts]
// Update the page title
const updated = await page.update({
  title: 'Updated Title',
})

// Update icon and cover
const styled = await page.update({
  icon: 'âœ¨',
  cover: 'https://example.com/new-cover.jpg',
})

// Lock the page to prevent editing
const locked = await page.update({
  isLocked: true,
})
```

## Render page content

Convert the page content to Markdown using `page.render()`:

```ts [render-page.ts]
const markdown = await page.render()
console.log(markdown)
// Output:
// # Welcome
//
// This is the introduction to our project.
//
// ## Next Steps
//
// - Read the documentation
// - Try the examples
```

This is useful for exporting content, generating documentation, or creating backups.

## Archive and restore pages

### Archive a page

Archiving moves the page to the trash:

```ts [archive-page.ts]
await page.archive()
console.log('Page archived')
```

### Restore a page

Restore an archived page:

```ts [restore-page.ts]
await page.restore()
console.log('Page restored')
```

## Clear page content

Remove all blocks from a page while keeping the page itself:

```ts [clear-page.ts]
// Clear all content
await page.clear()

// Add new content
await page.append([
  paragraph('Fresh start!'),
])
```

This is useful when you want to replace all content without creating a new page.
